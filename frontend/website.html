Architecture

Five layers. Zero ambiguity.

Every request flows through a deterministic pipeline. No LLM can hallucinate its way past the schema validation gate.

01
⬡
Interface Layer
FastAPI with API key auth, rate limiting per-key, CORS from env. Every request validated before touching business logic.
02
⋈
Orchestration Layer
DAG executor reads YAML workflow, builds execution plan, dispatches Celery tasks. State machine: PENDING → ACTIVE → VERIFIED → COMMITTED.
03
⚡
Execution Layer
Celery workers call your LLM (Anthropic, OpenAI, Ollama). JSON enforced on every response. Retry with exponential backoff.
04
◈
Validation Gate
Pydantic + JSON Schema + assertions on every output. Nothing advances the DAG on bad output. Period.

Developer Experience

Code that makes sense

Agent Contract (YAML)
# agents/engineering/ui-engineer.yaml
id: engineering.ui-engineer
name: UI Engineer
version: 2.1.0
division: engineering

input_schema:
  type: object
  required: [component_spec, tech_stack]
  properties:
    component_spec:
      type: string

output_schema:
  type: object
  required: [component_code, tests, story]

failure:
  max_retries: 3
  retry_strategy: exponential_backoff

success_metrics:
  assertions:
    - field: data.component_code
      rule: min_length:50
Python SDK
from syndicate.sdk import SyndicateClient

# Type-safe client
client = SyndicateClient(
    api_key="sk-your-key",
)

# Execute and wait
result = client.run_and_wait(
    "startup-mvp",
    context={"brief": "B2B SaaS"},
)

# Or stream in real-time
for event in client.stream(
    result.execution_id
):
    if event.type == "step.completed":
        print(f"✓ {event.data['step_name']}")

# CLI equivalent
# $ syndicate run startup-mvp \
#     --context '{"brief":"B2B"}' \
#     --wait
