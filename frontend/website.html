Architecture

Five layers. Zero ambiguity.

Every request flows through a deterministic pipeline. No LLM hallucinates past the validation gate.

01
⬡
Interface Layer
FastAPI with API key auth, per-key rate limiting, CORS from environment. Every request validated before touching business logic.
FastAPI · slowapi
02
⋈
Orchestration Layer
DAG executor reads YAML, builds execution plan, dispatches Celery tasks. State machine: PENDING → ACTIVE → VERIFIED → COMMITTED.
Celery · Redis
03
⚡
Execution Layer
Celery workers call your LLM. JSON enforced on every response. Exponential backoff on failure. Escalation after max retries.
Pluggable LLM
04
🛡️
Validation Gate
Pydantic + JSON Schema + success assertions on every output. Nothing advances the DAG on bad output. This is the key differentiator.
Hard Gate
05
💾
Memory Layer
Redis hot context (sub-ms reads, TTL) for cross-step flow. PostgreSQL WAL for durable audit trails. Step N reads Step 1's validated output.
Redis · PostgreSQL

Developer Experience

YAML contracts. Python SDK.

Readable by your whole team. Executed by a production-grade runtime. No framework lock-in.

Agent Contract — YAML
# agents/engineering/ui-engineer.yaml
id: engineering.ui-engineer
version: 2.1.0
division: engineering

input_schema:
  type: object
  required: [component_spec, tech_stack]

output_schema:
  type: object
  required: [component_code, tests]

failure:
  max_retries: 3
  retry_strategy: exponential_backoff

success_metrics:
  assertions:
    - field: data.component_code
      rule: min_length:50
Python SDK
from syndicate.sdk import SyndicateClient

# Type-safe, connection-pooled
client = SyndicateClient(
    api_key="sk-your-key"
)

# Execute + wait
result = client.run_and_wait(
    "startup-mvp",
    context={"brief":"B2B SaaS"}
)

# Or stream live events
for event in client.stream(
    result.execution_id):
    if event.type=="step.completed":
        print(f"✓ {event.data['step_name']}")
